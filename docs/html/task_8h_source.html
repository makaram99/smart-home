1182  * THIS FUNCTION MUST NOT BE USED FROM APPLICATION CODE. IT IS ONLY

1185  *
1186  * Called from the real time kernel tick (either preemptive or cooperative),

1192 
1193 /*
1194  * THIS FUNCTION MUST NOT BE USED FROM APPLICATION CODE. IT IS AN
1195  * INTERFACE WHICH IS FOR THE EXCLUSIVE USE OF THE SCHEDULER.
1196  *
1197  * THIS FUNCTION MUST BE CALLED WITH INTERRUPTS DISABLED.
1198  *
1199  * Removes the calling task from the ready list and places it both
1200  * on the list of tasks waiting for a particular event, and the
1201  * list of delayed tasks. The task will be removed from both lists
1202  * and replaced on the ready list should either the event occur (and
1203  * there be no higher priority tasks waiting on the same event) or
1204  * the delay period expires.
1205  *
1206  * @param pxEventList The list containing tasks that are blocked waiting
1207  * for the event to occur.
1208  *
1209  * @param xTicksToWait The maximum amount of time that the task should wait
1210  * for the event to occur. This is specified in kernel ticks,the constant
1211  * portTICK_RATE_MS can be used to convert kernel ticks into a real time
1212  * period.
1213  */
1214 void vTaskPlaceOnEventList( const xList * const pxEventList, portTickType xTicksToWait ) PRIVILEGED_FUNCTION;
1215 
1216 /*
1217  * THIS FUNCTION MUST NOT BE USED FROM APPLICATION CODE. IT IS AN
1218  * INTERFACE WHICH IS FOR THE EXCLUSIVE USE OF THE SCHEDULER.
1219  *
1220  * THIS FUNCTION MUST BE CALLED WITH INTERRUPTS DISABLED.
1221  *
1222  * This function performs nearly the same function as vTaskPlaceOnEventList().
